Architecture & Configuration Réseau

Cette page présente les notions fondamentales de l’architecture et de la configuration réseau, notamment les modèles client/serveur et poste à poste, ainsi que les concepts essentiels : adresse IP, masque de réseau, DNS et adresse MAC.

Architectures : Client/Serveur – Poste à poste

L’architecture réseau définit la manière dont les équipements communiquent et partagent les ressources. On distingue principalement deux modèles :

  • Architecture Client/Serveur : modèle centralisé fournissant des services à plusieurs clients.
  • Architecture Poste à poste : modèle décentralisé sans serveur central.

Adresse IP

L’adresse IP est un identifiant numérique unique permettant d’identifier un appareil sur un réseau.

  • Types : IPv4 et IPv6.
  • Rôle : Routage des données.
  • Attribution : Statique ou dynamique.

Masque de réseau

Le masque de réseau sépare la partie réseau et la partie hôte d’une adresse IP.

  • Exemple : 192.168.1.10 / 255.255.255.0
  • Utilité : Définition du sous-réseau.
  • CIDR : /24

DNS

Le DNS permet de traduire les noms de domaine en adresses IP.

  • Fonction : Résolution des noms.
  • Types : Récursif et autoritatif.
  • Importance : Navigation simplifiée.

Adresse MAC

L’adresse MAC identifie physiquement une interface réseau.

  • Format : 48 bits.
  • Niveau : Couche liaison de données.
  • Différence : Fixe vs variable (IP).

Pour approfondir ces notions, consultez les RFC et les cours en réseaux informatiques.
